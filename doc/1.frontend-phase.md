## ✅ Phase F0 — Khung dự án

React **18.3.1** và Tailwind **v3.4.19** — đã ghim version có chủ đích. Template
Vite mặc định cài React 19 + Tailwind v4, trái đặc tả, nên phải gỡ và cài lại.
Đừng chạy `npm update` bừa làm nhảy lên bản mới.

Đã cài đủ: Zustand, TanStack Query, React Router 6, React Hook Form, Recharts,
Lucide React, Axios.

Đã có sẵn:
- Cấu trúc thư mục theo feature (mục 8 của CLAUDE.md), thư mục rỗng giữ bằng
  `.gitkeep`.
- `src/index.css` — 13 CSS variable màu + bộ bo góc `--r/--r-lg/--r-xl`.
- `tailwind.config.js` — map token sang Tailwind, shadow 2 lớp cho card 2.5D,
  breakpoint chỉ `sm` 360px và `lg` 1180px (không làm tablet).
- `vite.config.js` — proxy `/api` sang cổng 8080, nên dev không cần CORS.
- `src/services/api.js` — axios tự gắn JWT, tự refresh khi 401; nhiều request
  401 đồng thời dùng chung một lần refresh.
- `src/utils/fmt.js` — format tiền VNĐ (số nguyên), số lượng 3 chữ số, ngày giờ.

**Đã kiểm chứng:** build production chạy được, và đã grep file CSS đầu ra xác
nhận token thực sự vào chứ không chỉ khai báo suông. Dev server lên trong ~90ms,
trả HTTP 200.

⚠️ `src/App.jsx` hiện chỉ là **trang tạm** hiển thị bảng màu để kiểm tra design
system. Phải thay bằng router thật ở Phase F1.

## ⬜ Phase F1 — Đăng nhập (làm tiếp theo)

Phụ thuộc backend Phase 2. Trước khi backend xong vẫn làm được phần giao diện
và store, chỉ chưa gọi API thật được.

Cần: Zustand store cho auth, trang đăng nhập, màn hình bắt đổi mật khẩu khi
`must_change_password = true`, route guard, và router thật thay cho `App.jsx`
tạm.

Theo CLAUDE.md mục 9, màn hình Đăng nhập thuộc nhóm hiệu ứng 2.5D **mạnh** vì là
ấn tượng đầu tiên.
