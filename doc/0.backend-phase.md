## ✅ Phase 0 — Cấu hình project

- Package `com.brewdesk.app`, class chính `BrewDeskApplication`.
- Spring Boot 4.0.7 + Java 25. Bản 3.3.x **không** dùng được vì không hỗ trợ
  Java 25.
- Dependency đã thêm ngoài template: `jjwt` 0.13.0 (api/impl/jackson),
  `springdoc-openapi-starter-webmvc-ui` 3.0.0, `aspectjweaver`.
  Boot 4 đã bỏ `spring-boot-starter-aop` nên phải khai báo AspectJ trực tiếp.
- `artifactId` là `brewdesk`, jar build ra `brewdesk-0.0.1-SNAPSHOT.jar`.

**Đã kiểm chứng:** app boot trong ~2 giây, HikariCP kết nối PostgreSQL 16.14
thật, Tomcat lên cổng 8080.

## ✅ Phase 1 — Database

Flyway đang ở **version 2**. Database có đúng 25 bảng, 27 CHECK constraint,
40 khóa ngoại, 55 index.

⚠️ **KHÔNG sửa `V1__init.sql` và `V2__seed.sql`** — hai file đã apply vào DB,
sửa dù một ký tự comment cũng làm sai checksum khiến app không boot. Cần thay
đổi gì thì viết `V3__*.sql` mới.

**Đã kiểm chứng bằng test thật** (insert dữ liệu sai, kỳ vọng DB từ chối):
15/15 trường hợp đúng. DB chặn mức ngọt 75%, tồn kho âm, giá âm, combo chứa
chính nó, điểm QC ngoài thang 1–5, tick checklist trùng ngày, role ngoài
ADMIN/STAFF, thanh toán ngoài CASH/TRANSFER.

Một quy tắc cố ý **không** đặt ở DB: `discount_value` vượt `subtotal` thì DB cho
qua, vì logic PERCENT và FIXED khác nhau — phải chặn ở Service layer.

Dữ liệu seed: 3 ca (P1/P2/P3), 10 đơn vị tính (g→kg, ml→l hệ số 0.001), 10 nhóm
nguyên liệu, 6 variant (3 mức ngọt + 3 mức đá), 1 tài khoản `admin` mật khẩu
`admin123` với `must_change_password = true`. Hash bcrypt đã verify khớp.

## ⬜ Phase 2 — `common/` + `auth/` (làm tiếp theo)

### Cần làm

`common/dto/` — `ApiResponse<T>` và `PageResponse<T>` theo đúng format ở CLAUDE.md
mục 4: `{success, data, message, errorCode}`.

`common/exception/` — `AppException` mang `errorCode`, và
`GlobalExceptionHandler` map mọi lỗi về đúng format trên.

`common/security/` — `JwtUtil` (phát hành + verify), `JwtFilter`,
`UserDetailsServiceImpl`.

`common/config/` — `SecurityConfig` (mở `/api/v1/auth/**`, Swagger, còn lại
yêu cầu token), `SwaggerConfig`, `CorsConfig`.

`common/audit/` — `AuditLog` entity + `AuditAspect` dùng `@Around`.

`auth/` — 4 endpoint: `POST /api/v1/auth/login`, `/refresh`, `/change-password`,
và `POST /api/v1/admin/staff` (chỉ ADMIN).

### Lưu ý bắt buộc

- **`JWT_SECRET` hiện chưa được validate** vì chưa có class nào đọc nó. Khi viết
  `JwtUtil` phải kiểm tra secret tồn tại và dài tối thiểu 32 ký tự, thiếu thì
  cho app fail ngay lúc khởi động.
- Tài khoản có `must_change_password = true` phải bị chặn dùng API khác cho tới
  khi đổi mật khẩu xong.
- Mật khẩu bcrypt, không bao giờ lưu plaintext.

### Coi như xong khi

- Đăng nhập bằng `admin` / `admin123` trả về access token và refresh token.
- Gọi endpoint cần quyền mà không có token thì trả 401 đúng format JSON, không
  phải trang lỗi mặc định của Spring.
- `/swagger-ui.html` mở được (hiện đang trả 401 vì chưa có `SecurityConfig`).
- Refresh token đổi được access token mới.
