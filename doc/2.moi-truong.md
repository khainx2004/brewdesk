# Môi trường chạy — thiết lập và cạm bẫy đã gặp

Cập nhật lần cuối: 19/07/2026

## Chạy nhanh

```bash
# 1. Database
docker compose up -d

# 2. Backend — PHẢI export biến môi trường trước
set -a && . ./.env && set +a
cd source/backend && ./mvnw spring-boot:run     # cổng 8080

# 3. Frontend
cd source/frontend && npm run dev               # cổng 5173
```

Đăng nhập: `admin` / `admin123` (bị bắt đổi mật khẩu lần đầu).

## Cạm bẫy đã gặp — đừng mất thời gian chẩn đoán lại

**Lỗi `password authentication failed for user "brewdesk_user"`**
Gần như luôn là do quên export biến môi trường, không phải sai mật khẩu DB.
`application.yml` cố ý không đặt giá trị mặc định cho `DB_PASSWORD` và
`JWT_SECRET` để app fail ngay thay vì chạy với credential lộ trong source.
Spring Boot **không** tự đọc file `.env`.

**`./mvnw spring-boot:run` in `BUILD SUCCESS` kể cả khi app crash lúc khởi động.**
Phải đọc log chứ đừng tin exit code. Đừng dùng exit code này trong script deploy.

**Máy chỉ có JRE thì không compile được.** Cần gói `-devel` để có `javac`:
`sudo dnf install java-25-openjdk-devel`.

**Podman rootless cần bật socket trước:**
`systemctl --user enable --now podman.socket`, rồi `podman compose up -d`.

**Cẩn thận khi dừng app bằng `pkill`.** Pattern `spring-boot:run` khớp luôn dòng
lệnh của chính script đang chạy nên nó tự giết mình. Dùng `pkill -f
"[s]pring-boot:run"` và **không** đặt chung lệnh khởi động lại trong cùng một
dòng, vì chuỗi `spring-boot:run` xuất hiện lại sẽ làm bracket trick vô hiệu.

**Sửa migration đã apply làm hỏng Flyway checksum.** Đã dính một lần: file
`V1__init.sql` rỗng bị apply nhầm, phải xóa dòng trong bảng
`flyway_schema_history` mới ghi lại schema thật được.

## Sửa file rồi mà app không đổi?

`spring-boot:run` đang chạy **không** tự nạp lại file trong
`src/main/resources` (ví dụ migration mới). Phải khởi động lại tiến trình.

## Git

Repo: https://github.com/khainx2004/brewdesk (public), remote qua SSH.
Danh tính git đặt riêng trong repo, dùng email ẩn của GitHub.
Commit **không** kèm dòng Co-Authored-By.

Cố ý giữ ngoài repo: `.env` (chỉ commit `.env.example`) và
`.claude/settings.local.json`.

Về cấu hình Claude Code: `.claude/settings.json` là bản dùng chung, có trong
repo. `.claude/settings.local.json` là của riêng từng máy — nó tích lũy đường
dẫn cá nhân, session ID và có thể lẫn cả credential — bản trên máy dev từng chứa
nguyên một lệnh `psql` kèm mật khẩu database. Tuyệt đối không commit. Máy này còn
có thêm một quy tắc chặn nó ở gitignore toàn cục `~/.config/git/ignore`.
